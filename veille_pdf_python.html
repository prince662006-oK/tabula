pdfplumber v0.11+

pdfplumber est construit sur pdfminer.six et donne accès à chaque élément d'une page PDF avec ses coordonnées précises (x0, y0, x1, y1). C'est le meilleur outil pour comprendre la structure d'un document avant d'en extraire les données.

coords

Retourne les coordonnées exactes de chaque mot, ligne et caractère sur la page.

find_tables

Détecte automatiquement les tableaux et retourne leur bounding box.

crop()

Permet de découper une zone précise de la page pour une extraction ciblée.

100% Python

Pas de dépendance Java. Installation simple avec pip, fonctionne partout.

zone_ciblee.py pdfplumber 
import pdfplumber

with pdfplumber.open("rapport.pdf") as pdf:
    page = pdf.pages[0]

    # 1. Trouver le mot-clé "Flux de trésorerie"
    mots = page.extract_words()
    y_debut = None

    for mot in mots:
        if 'trésorerie' in mot['text'].lower():
            y_debut = mot['y1']  # juste après le titre
            break

    if y_debut:
        # 2. Découper la page : seulement la zone du tableau
        # crop(x0, top, x1, bottom)
        zone = page.crop((
            0,            # x gauche
            y_debut + 5,  # top (après le titre)
            page.width,   # x droite
            page.height   # bas de page
        ))

        # 3. Extraire le tableau dans cette zone précise
        table = zone.extract_table()
        print(f"Tableau extrait : {len(table)} lignes")
        for row in table:
            print(row)

camelot-py v0.11+

Camelot est la bibliothèque de référence pour l'extraction de tableaux complexes. Il propose deux modes d'extraction complémentaires — Lattice (lignes visibles) et Stream (espaces blancs) — avec un score de confiance intégré pour valider la qualité de l'extraction.

Lattice

Suit les lignes visibles du tableau. Parfait pour les bilans comptables avec bordures.

Stream

Analyse les espaces blancs. Pour les tableaux sans bordures (états financiers).

accuracy

Score de confiance (0–100%) sur chaque extraction. Permet de filtrer les mauvais résultats.

MultiPage

Traitement batch : extrait tous les tableaux d'un PDF en une seule ligne de code.

traitement_batch.py camelot 
import camelot
import pandas as pd
import os

def extraire_tous_les_tableaux(dossier_pdf: str) -> dict:
    """
    Extrait tous les tableaux de tous les PDFs d'un dossier.
    Retourne un dict {nom_fichier: [DataFrame, ...]}
    """
    resultats = {}

    for fichier in os.listdir(dossier_pdf):
        if not fichier.endswith(".pdf"):
            continue

        chemin = os.path.join(dossier_pdf, fichier)

        try:
            # Extraire toutes les pages d'un coup
            tables = camelot.read_pdf(chemin, pages="all", flavor="lattice")

            # Filtrer : garder seulement les tableaux avec précision > 80%
            bons_tableaux = [
                t.df for t in tables
                if t.accuracy > 80 and not t.df.empty
            ]

            resultats[fichier] = bons_tableaux
            print(f"✓ {fichier} : {len(bons_tableaux)}/{len(tables)} tableaux retenus")

        except Exception as e:
            print(f"✗ {fichier} : erreur — {e}")

    return resultats

# Utilisation
data = extraire_tous_les_tableaux("./rapports/")

# Consolider dans un seul Excel
with pd.ExcelWriter("extraction_complete.xlsx") as writer:
    for nom_pdf, tableaux in data.items():
        for i, df in enumerate(tableaux):
            sheet = f"{nom_pdf[:20]}_{i+1}"
            df.to_excel(writer, sheet_name=sheet, index=False)

tabula-py v2.9+

tabula-py est un wrapper Python autour de tabula-java, un outil Java éprouvé. Son atout majeur est la vitesse : il excelle sur les documents standardisés et volumineux comme les relevés bancaires ou les rapports répétitifs. Requiert Java 8+ sur le système.

Vitesse

3 à 5× plus rapide que Camelot. Optimal pour traiter des centaines de pages identiques.

guess=True

Détection automatique de la zone du tableau. Fonctionne sans configuration sur docs simples.

area

Paramètre pour cibler une zone précise (top, left, bottom, right) en points PDF.

Java

Basé sur tabula-java, projet open-source mature. Requiert JRE 8+ installé.

gros_volumes.py — 500 pages de relevés tabula-py 
import tabula
import pandas as pd
import time

def extraire_releves_rapide(pdf_path: str) -> pd.DataFrame:
    """
    Extrait rapidement tous les relevés d'un PDF volumineux.
    tabula traite l'intégralité en une seule invocation Java.
    """
    debut = time.time()

    # Extraction de TOUTES les pages en une passe
    # tabula.read_pdf avec pages="all" est très efficace
    dfs = tabula.read_pdf(
        pdf_path,
        pages="all",
        multiple_tables=True,
        guess=True,
        silent=True,
        pandas_options={"header": 0}
    )

    elapsed = time.time() - debut
    print(f"⚡ {len(dfs)} tableaux extraits en {elapsed:.1f}s")

    # Consolider tous les tableaux de même structure
    if not dfs:
        return pd.DataFrame()

    # Garder seulement les df avec les mêmes colonnes que le premier
    ref_cols = set(dfs[0].columns)
    compatibles = [df for df in dfs if set(df.columns) == ref_cols]

    # Fusion en un seul DataFrame
    df_final = pd.concat(compatibles, ignore_index=True)
    df_final = df_final.dropna(how="all")

    print(f"✓ {len(df_final)} lignes consolidées")
    return df_final

# Sur 500 pages de relevés bancaires :
df = extraire_releves_rapide("releves_2024_complet.pdf")
df.to_excel("releves_consolides.xlsx", index=False)

combinaison_optimale.py — les 3 bibliothèques ensemble BEST PRACTICE 
import pdfplumber
import camelot
import tabula
import pandas as pd

KEYWORDS_COMPLEXES = ["bilan", "flux", "trésorerie", "résultat", "annexe"]

def smart_extract(pdf_path: str, page_num: int) -> pd.DataFrame:
    """
    Pipeline intelligent : chaque outil fait ce qu'il sait faire.
    """

    # ─── PHASE 1 : pdfplumber analyse la structure ───────────────
    with pdfplumber.open(pdf_path) as pdf:
        page = pdf.pages[page_num - 1]
        texte = (page.extract_text() or "").lower()
        tableaux = page.find_tables()

        if not tableaux:
            print(f"Page {page_num} : aucun tableau détecté")
            return pd.DataFrame()

        bbox = tableaux[0].bbox  # coordonnées du tableau

    # ─── PHASE 2 : décision d'aiguillage ─────────────────────────
    est_complexe = any(kw in texte for kw in KEYWORDS_COMPLEXES)

    # ─── PHASE 3a : Camelot pour les tableaux complexes ──────────
    if est_complexe:
        print(f"Page {page_num} → Camelot (complexe)")
        tables = camelot.read_pdf(pdf_path, pages=str(page_num), flavor="lattice")

        if tables and tables[0].accuracy > 70:
            return tables[0].df

        # Fallback : stream si lattice échoue
        tables = camelot.read_pdf(pdf_path, pages=str(page_num), flavor="stream")
        if tables:
            return tables[0].df

    # ─── PHASE 3b : Tabula pour les tableaux simples ─────────────
    else:
        print(f"Page {page_num} → Tabula (rapide)")
        dfs = tabula.read_pdf(pdf_path, pages=page_num, multiple_tables=True, silent=True)
        if dfs:
            return dfs[0]

    return pd.DataFrame()

# Test
df = smart_extract("rapport_financier.pdf", page_num=3)
print(df)
